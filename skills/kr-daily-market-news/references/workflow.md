# Daily Market News Workflow

## Source Priority

1. Use domestic media RSS articles as the only default source for market and sector body links.
2. Use Google News RSS only as a query-based discovery signal for ranking, theme detection, and identifying which sectors/keywords are active.
3. Do not render Google News RSS wrapper URLs in `marketNews`, `sectorNews`, Naver post link cards, or the `출처` table.
4. Keep any supplied market-level official links and RSS article links separate in the JSON and Markdown.
5. Do not add watchlist company DART/KRX/IR links in daily market-news output.

## Watchlist Format

The default watchlist path is `examples/kr/daily-watchlist.json`.

```json
[
  { "ticker": "005930", "name": "삼성전자", "keywords": ["HBM", "실적"] }
]
```

Each item requires a six-digit `ticker` and `name`. The daily market-news collector may validate this file for compatibility, but it does not collect watchlist stock news or watchlist company disclosure links. Those belong in a separate stock/watchlist workflow.

## Sector Stock Seeds

The default sector stock seed path is `examples/kr/daily-sector-stocks.json`.

It is keyed by the fixed 10 daily-market sectors. Each sector keeps 20 representative stocks with `ticker`, `name`, and `keywords`. Live sector news collection builds one Google News RSS discovery query from the sector label, representative stock names, and keywords. Actual `sectorNews` items come only from domestic RSS articles whose title or description matches the sector label, representative stock names, or keywords. Keep at most five items per sector after URL/headline dedupe.

If the seed file is missing or a sector has no stocks, the collector falls back to broad sector-label matching against the domestic RSS article pool.

## Scheduled Automation Prompt

Recommended Codex Desktop Automation prompt:

```text
Use $kr-daily-market-news to create today's Korean daily market-news report for blog publication.

Scope:
- Market-wide and sector report only.
- Do not include watchlist stock-news sections or watchlist DART/KRX links.
- Use Korean market news flow from domestic RSS article URLs and keep source URLs exact.
- Write artifacts to analysis-example/kr-market/daily-news-YYYY-MM-DD.md and .json.
- Then use $kr-naver-blog-publish to prepare and publish the report to Naver Blog in scheduled mode.

Safety:
- Do not publish if login, CAPTCHA, source collection, body generation, image upload, duplicate same-day post check, disclaimer check, or source URL validation fails.
- If publication fails, leave the draft artifacts and report the failure in the automation run.
```

## Operating Rules

- Do not invent market moves or company events. If live collection returns too little material, render an explicit low-source warning instead of filling gaps.
- Keep exact source URLs in the JSON and Markdown.
- For live collection, keep only articles dated the same day as `asOfDate`; do not backfill with older or undated articles when fewer than five same-day items are available.
- In the Naver post, keep collection warnings as short end notes under `### 수집 참고`; do not put a `## 수집 경고` block at the top.
- In the Naver `시장 주요 뉴스` section, use raw URL link-card lines (`1. 제목` then `https://...`) instead of Markdown bullet links.
- Do not use Markdown bullet lines anywhere in the Naver post. Use numbered lines, tables, or short paragraphs instead.
- Treat RSS news as secondary reporting, not primary disclosure evidence.
- The blog manifest generated by this skill sets `automation.scheduledPublishAllowed` only for daily market-news reports. It does not change the default manual-approval publishing path for stock memos.
